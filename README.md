# 🔋 SafeCell — AI 배터리 열폭주 예측 경보 시스템

> 전기차 충전소·지하주차장의 배터리 **열폭주(thermal runaway)를 발화 평균 8분 전에 예측**하고,
> 119에 자동 연동하는 멀티센서 기반 AI 경보 시스템입니다.

## 🚨 문제 (Problem)

배터리 열폭주는 일반 연기·열감지기가 반응할 때면 이미 진화가 불가능한 단계입니다.
발화 후 전소까지 수 분, 유독가스와 재발화로 밀폐된 지하 공간에서 곧장 인명피해로 이어집니다.

- 전기차 보급 급증 + 지하 충전 인프라 의무화
- 전기차 화재보험 기본료 **2023~2025년 약 40% 상승**
- 지하주차장·충전소 관리자는 화재 1건당 민·형사 책임과 수억 원 배상 리스크 부담

기존 단순 연기·열감지기는 **사후 탐지**에 그쳐, 발화 전 예측이라는 본질적 해법이 필요합니다.

---

## 💡 솔루션 (Solution)

5종 멀티센서를 엣지 디바이스에서 융합하고, 정상 패턴만 학습한 **LSTM-AE 비지도 이상탐지**로
'발화 전조'를 포착합니다.

```
[멀티센서] → [엣지 AI 추론] → [3단계 경보] → [119 자동 연동]
  가스/VOC·HF       LSTM-AE        Green
  열화상            이상탐지        Yellow  →  관리자 알림
  온도                             Orange  →  119 자동 신고
  전압
  음향
```

- **3단계 경보**: Green(정상) → Yellow(주의) → Orange(위험)
- **멀티모달 퓨전**: 2개 이상 센서 동시 이상 시에만 경보 → 오탐 억제
- **119 자동 연동**: BACnet/IP 표준 API 경유

---

## 🧠 핵심 기술 (Tech)

| 구분 | 내용 |
|------|------|
| 이상탐지 모델 | LSTM-AE (정상 데이터만 학습하는 비지도 시계열 모델) |
| 센서 융합 | 가스·VOC·HF / 열화상 / 온도 / 전압 / 음향 5종 멀티모달 |
| 데이터 생성 | PyBaMM (P2D 전기화학 시뮬레이션) + Chen2020 파라미터셋 |
| 데이터 증강 | SMOTE + TimeGAN |
| 엣지 추론 | 경량화 모델, 실시간 추론 |

**왜 LSTM-AE인가?** 발화 데이터는 극히 희소해 지도학습이 불가능합니다.
LSTM-AE는 정상 데이터만으로 학습해 미세한 패턴 이탈을 탐지하므로 이 문제의 본질적 해법입니다.

## ▶️ 실행 방법 (Getting Started)

```bash
# 1. 저장소 클론
git clone https://github.com/kwanghoon6703-lgtm/safecell.git
cd safecell

# 2. 의존성 설치
pip install -r requirements.txt

# 3. 데이터 생성 (PyBaMM 시뮬레이션)
python src/generate_data.py

# 4. 모델 학습
python src/train_lstm_ae.py

# 5. 대시보드 실행
uvicorn src.app:app --reload   # 백엔드
cd dashboard && npm install && npm run dev   # 프론트엔드
```

> 해커톤 진행에 따라 코드가 순차적으로 추가됩니다.
